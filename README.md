    ㄴ강의후기: 프로젝트 환경설정부터 간단한 비즈니스 로직이 담긴 웹 애플리케이션을 직접 만들어보며 웹 MVC와 스프링 DB를 익힐 수 있었고, 단위 테스트와 통합 테스트까지 진행하며 백엔드 개발자들의 전반적인 역할을 엿볼 수 있었다. 사실, 작년 우리FISA 교육을 듣지 않았더라면 이 과정도 어렵게 느껴졌을 것 같다. 그때 열심히 공부한 덕분에 강의를 들으면서 그때 학습했던 것들이 자연스럽게 복습되는 느낌이었다.  
    프론트엔드 개발만 할 때는 객체지향의 진정한 의미를 체감하기 어려웠는데, 스프링을 조금만 다뤄보아도 그 철학이 명확히 다가왔다. 특히 인터페이스를 통해 기존 코드를 변경하지 않고 구현체를 바꿔 낄 수 있는 유연성이 굉장히 매력적이었다.
    또한, JDBC → JdbcTemplate → JPA → Spring Data JPA로 이어지는 진화 과정을 직접 경험하면서, 마지막에는 인터페이스만으로 DB 접근과 데이터 조회가 자동으로 이루어지는 순간은 정말 경이로웠다. 스프링 부트와 JPA만 잘 공부하면 나도 풀스택 개발자 아니 1인 개발자가 되어서 뭐든 다 혼자 할 수 있을지도..?
